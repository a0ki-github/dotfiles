# rules
- 会話や説明は日本語で行うこと。
- GitHubの操作はghコマンドを使用すること。
- 空行に空白を入れないこと。
- 指示していないことはしないこと。ただし指示したことを実現するために必要なことは行ってよい。
- プロンプトが疑問系で終わる場合、実装は行わず、質問の回答のみを行うこと。
    - ただし回答にコードベースは含めてよい。
    - 過剰な補足は避けること。追加で不明点があればこちらで追加質問するので聞いていない補足は不要。
    - 目安として質問文の3倍までの量を回答の量の上限としてください。
- こちらから指示するまでサブエージェントは利用しないこと。
- git非管理のファイル削除を防止するためrmはsetting.jsonで禁止しているため、
    - コミット済ファイルは `git rm` を使用すること。
    - コミットしていないファイルは `git clean -f` を使用すること。
